# VoiceTrace AI

생성형 AI 영상에서 인물 음성이 얼마나 자연스럽게 들리는지 진단하는 제작자용 품질 분석 프로토타입입니다.

## 프로젝트 요약

VoiceTrace AI는 AI 생성 영상 또는 음성 파일의 품질을 HNR(Harmonics-to-Noise Ratio), HF ratio 같은 음성 신호 지표로 분석하고, 제작자가 이해하기 쉬운 리포트로 정리하는 도구를 목표로 합니다. 딥페이크 여부를 단정하는 앱이 아니라, 업로드 전 자연스러움·잡음·고주파 거칠음 같은 리스크를 점검하는 Creator Preflight Report에 가깝습니다.

본 프로토타입은 KCC2026 학부생 논문 부문 Poster 발표 논문으로 Accept된 `한국어 AI 동영상 생성 음성과 인간 발화의 음성 신호 차이 분석: HNR과 HF ratio를 중심으로` 연구 경험을 바탕으로 기획했습니다.

## 실행 방법

데모 샘플만 확인할 때는 별도 설치가 필요 없습니다.

1. 이 폴더의 `index.html`을 브라우저로 엽니다.
2. 데모 샘플을 선택합니다.
3. `분석 리포트 생성` 버튼을 누릅니다.

실제 파일 분석까지 확인하려면 FastAPI 서버를 먼저 실행합니다.

```bash
cd ..\server
pip install -r requirements.txt
uvicorn app.main:app --reload
```

그 다음 `web-demo/index.html`에서 서버 주소를 `http://127.0.0.1:8000`으로 두고 오디오/비디오 파일을 선택한 뒤 `분석 리포트 생성`을 누릅니다.

## 포함 파일

- `index.html`: 대시보드 화면
- `style.css`: 화면 스타일
- `app.js`: 데모 분석, FastAPI 업로드 연동, 그래프 렌더링
- `proposal.md`: 프로젝트 기획서 본문

## 프로토타입 범위

현재 버전은 과제 제출용 웹 데모입니다. 파일을 선택하지 않으면 데모 샘플로 흐름을 보여주고, FastAPI 서버가 실행 중이면 선택한 오디오/비디오 파일을 `/analyze`로 업로드해 실제 신호 지표를 표시합니다. 모바일 제품 버전은 React Native/Expo 앱에서 같은 서버 API를 호출합니다.

## 모바일 앱 확장 계획

- React Native/Expo 기반 Android 앱 구현
- `expo-document-picker` 기반 오디오/비디오 파일 선택
- FastAPI 서버로 파일 업로드
- Python 분석 서버에서 FFmpeg로 오디오 추출
- HNR/HF ratio, 스펙트럼 특징, 잡음 리스크 계산
- 분석 결과 JSON을 앱에서 카드와 그래프로 표시
- EAS Build로 테스트용 APK와 Google Play 제출용 AAB 생성

## 향후 확장

- 인간 발화 기준 샘플과 자동 비교
- 모델별 AI 생성 음성 품질 비교 리포트
- 텍스트 생성 AI를 활용한 제작자용 개선 가이드 자동 작성

## 한계와 안전 문구

HNR과 HF ratio는 AI 생성 여부를 단독으로 증명하는 지표가 아닙니다. 본 프로젝트는 법적 증거나 100% 탐지 도구가 아니라, 제작자가 음성 품질을 점검하고 개선 방향을 잡기 위한 참고용 리포트를 제공합니다. 업로드된 음성/영상 파일은 민감한 데이터가 될 수 있으므로, 실제 서비스에서는 보관 기간, 삭제 정책, 제3자 처리 여부를 명확히 안내해야 합니다.
